
O NoSQL é uma abordagem que literalmente não usa o [[SQL]] como padrão abreviação de Not Only SQL, é uma abordagem de banco de dados que difere dos bancos de dados relacionais tradicionais, como o [[SQL]], ao não utilizar o modelo de dados tabular e a linguagem SQL para consultas. 

Em vez disso, os bancos de dados NoSQL são projetados para lidar com grandes volumes de dados não estruturados ou semi-estruturados, oferecendo flexibilidade e escalabilidade.

Os bancos de dados NoSQL são frequentemente usados em aplicações web e móveis que requerem alta disponibilidade e desempenho escalável. Eles podem ser categorizados em diferentes tipos, como bancos de dados de documentos, bancos de dados chave-valor, bancos de dados orientados a colunas e bancos de dados orientados a grafos.

Algumas das vantagens do NoSQL incluem a capacidade de lidar com grandes volumes de dados não estruturados, a escalabilidade horizontal mais fácil e a capacidade de lidar com mudanças frequentes nos requisitos da aplicação. 

No entanto, também há desafios associados ao uso do NoSQL, como a falta do suporte completo à transações [[ACID]] e a necessidade de um planejamento cuidadoso da estrutura dos dados.

No geral, o NoSQL oferece uma alternativa interessante aos bancos de dados relacionais tradicionais para certas aplicações que exigem flexibilidade e escalabilidade.

Exemplos de bancos de dados NoSql : 

- [[MongoDB]]: um banco de dados de documentos que armazena dados em documentos JSON sem esquema fixo.
- Cassandra: um banco de dados orientado a colunas projetado para lidar com grandes volumes de dados distribuídos.
- Redis: um banco de dados chave-valor que armazena dados na memória para acesso rápido.
- Neo4j: um banco de dados orientado a grafos
