## AWS Directory Service: O que é e quando usar?

O AWS Directory Service permite conectar seus recursos da AWS (como instâncias EC2) a um diretório corporativo existente ou criar um novo diretório na AWS. Basicamente, ele fornece gerenciamento centralizado de usuários, grupos e políticas de acesso para seus recursos na nuvem.

**Para que serve?**

* **Gerenciamento centralizado de identidades:** Crie e gerencie usuários, grupos e políticas de acesso em um único local.
* **Autenticação única (SSO):** Permita que os usuários acessem recursos da AWS e aplicativos conectados ao diretório usando as mesmas credenciais.
* **Integração com aplicações existentes:** Conecte aplicações baseadas em diretório (como Microsoft[[ Active Directory]]) à AWS.
* **Simplificação do gerenciamento de acesso:** Aplique políticas de acesso consistentes em toda a sua infraestrutura AWS.
* **Conformidade com políticas de segurança:** Implemente controles de acesso granulares para atender aos requisitos de segurança.


**Quando usar?**

* **Migração de aplicações para a AWS:** Se você possui aplicações que dependem de um diretório, o AWS Directory Service facilita a migração dessas aplicações para a nuvem.
* **Infraestrutura híbrida:** Conecte seus recursos na AWS ao seu diretório corporativo existente para um gerenciamento unificado de identidades.
* **Necessidade de SSO:** Simplifique o acesso do usuário a recursos da AWS com autenticação única.
* **Gerenciamento centralizado de usuários e grupos:**  Se você precisa gerenciar um grande número de usuários e grupos na AWS, o Directory Service oferece uma solução centralizada e eficiente.
* **Aplicações que requerem autenticação baseada em diretório:**  Use o Directory Service para fornecer autenticação para aplicações que exigem um diretório, como SharePoint ou aplicações .NET.


Em resumo, use o AWS Directory Service quando precisar de uma solução robusta e escalável para gerenciar identidades e controlar o acesso aos seus recursos na AWS, especialmente se você já utiliza um diretório ou precisa de funcionalidades como SSO e gerenciamento centralizado.
