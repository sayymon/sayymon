
Amazon RDS é um serviço de banco de dados relacional totalmente gerenciado pela Amazon Web Services (AWS). Ele facilita a configuração, operação e escalabilidade de bancos de dados relacionais na nuvem.

Este serviço é ideal para empresas que desejam ter um banco de dados confiável, escalável e de alto desempenho sem se preocupar com a infraestrutura subjacente. O Amazon RDS é amplamente utilizado por organizações de todos os tamanhos que precisam armazenar e acessar grandes volumes de dados de forma eficiente.

Quando utilizar o Amazon RDS:

1. Quando você precisa de um banco de dados relacional confiável e altamente disponível.
2. Quando deseja escalabilidade automática para lidar com picos de tráfego.
3. Quando precisa se concentrar no desenvolvimento de aplicativos em vez da administração do banco de dados.
4. Quando quer reduzir os custos operacionais e o tempo gasto na manutenção do banco de dados.

Em resumo, o Amazon RDS é uma excelente opção para empresas que buscam simplificar a gestão e otimizar o desempenho do seu banco de dados relacional na nuvem.

## Quais bancos posso configurar no RDS ? 


O Amazon RDS suporta vários tipos de bancos de dados relacionais, incluindo [[MySQL]], [[PostgreSQL]], [[MariaDB]], [[Oracle]] e [[Microsoft SQL Server]]. Você pode escolher o tipo de banco de dados que melhor atende às necessidades do seu aplicativo e configurá-lo facilmente no Amazon RDS. Além disso, o serviço oferece opções de instâncias otimizadas para diferentes tipos de carga de trabalho, garantindo alto desempenho e eficiência para o seu banco de dados relacional.

O Amazon RDS é um serviço gerenciado que facilita a execução de bancos de dados relacionais (como MySQL, PostgreSQL, Oracle, SQL Server e MariaDB) na AWS.  Você não precisa se preocupar com servidores, armazenamento, backups, etc. A AWS cuida disso para você.

**Para que serve?** Armazenar e acessar dados de forma estruturada e confiável em seus aplicativos.

**Quando utilizar?**

* Quando precisar de um banco de dados relacional fácil de configurar e gerenciar.
* Quando precisar de alta disponibilidade e escalabilidade.
* Quando quiser economizar tempo e recursos na administração do banco de dados.

**Principais conceitos:**

* **Instâncias:** Servidores virtuais que hospedam seu banco de dados.
* **Armazenamento:** Espaço em disco para seus dados.
* **Snapshots:** Backups pontuais do seu banco de dados.
* **Segurança:** Controle de acesso via grupos de segurança.
* **Monitoramento:** Métricas de desempenho e logs para acompanhar a saúde do banco de dados.


Em essência, o RDS simplifica a execução de bancos de dados na nuvem, permitindo que você se concentre no desenvolvimento de aplicativos.
