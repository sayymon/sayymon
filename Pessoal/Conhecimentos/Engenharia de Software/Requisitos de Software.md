
Os requisitos são as condições que o [[software]] precisa para ser considerado funcional e que atenda as demandas para qual ele esta sendo criado e as necessidades ou problemas associados.

Para levantar os requisitos de software precisamos primeiro ter em mente o dominio do contexto ou seja entender bem sobre o negócio que ele esta inserido para conseguir direcionar para uma solução adequada.

## Tipos de Requisitos

Temos dois principais tipos de requisitos os [[Requisitos de Software Funcionais]] e os [[Requisitos de Software não funcionais]] São a base da [[Engenharia de Software]]

Outro ponto importante é a identificação clara dos [[stakeholders]] que podem agregar mais ou menos em relação a alguns requisitos.
